# TD3

语言: 简体中文

TD3 和 SAC 共用同一套 off-policy 训练入口，但 owner 路径和超参数不同。

## 默认入口

```bash
uv run train --algo td3 --task g1_walk_flat --sim mujoco
```

## 适用场景

- off-policy 训练
- 与 SAC 共享同一脚本入口，但 owner 路径不同

## 运行模型

- 和 SAC 共用 shared-memory off-policy runner
- 常见用途是用同一任务 owner 做 SAC / TD3 对照
- 是否同步采集由 `training.no_sync_collection` 控制

## 实现脚本

TD3 顶层 CLI 会路由到 `scripts/train_offpolicy.py`，并选择
`conf/offpolicy/task/td3/` 下的 owner YAML。普通调参继续用 Hydra override；
算法、任务和后端选择放在 `--algo`、`--task`、`--sim`。

## 常见命令

```bash
uv run train --algo td3 --task g1_walk_flat --sim mujoco

uv run train --algo td3 --task g1_walk_flat --sim mujoco \
  training.no_play=true

uv run eval --algo td3 --task g1_walk_flat --sim mujoco \
  --load-run "2024-02-04_12-00-00"
uv run eval --algo td3 --task g1_walk_flat --sim mujoco \
  --load-run -1 --render-mode record training.export_onnx=false
```

如果只是做 off-policy 回放或录 MP4, 可以加 `training.export_onnx=false` 跳过
`policy.onnx` 导出。这个开关与 SAC / FlashSAC 共用, 仅在 `train_offpolicy.py`
链路生效；范围说明见
[评估、回放与恢复训练](../2-training/2-playback-and-resume.md)。

## 什么时候优先看 TD3

- 任务 owner 已经为 TD3 单独调过超参数
- 你需要和 SAC 做同任务对照
- 你想继续沿用同一套 off-policy 训练栈，但换成 TD3 owner
